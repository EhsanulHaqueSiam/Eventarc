### Import Wizard (`/events/$eventId/guests/import`)

```
+----------------------------------------------------------+
| [Breadcrumb: Events > Event Name > Guests > Import]      |
|                                                          |
| Import Guests                                            |
|                                                          |
| (1)----(2)----(3)----(4)----(5)                          |
| Upload  Map   Validate Dupes  Confirm                    |
|                                                          |
| +------------------------------------------------------+ |
| |                                                      | |
| |  [Step content area — varies per step]               | |
| |                                                      | |
| +------------------------------------------------------+ |
|                                                          |
|                          [Back]  [Next / Confirm Import] |
+----------------------------------------------------------+
```

**Step 1 — Upload:** Drag-and-drop zone with file type hint. Shows file name and row count after parse.
**Step 2 — Map Columns:** Table showing detected columns with dropdown selectors mapping to guest fields (Name, Phone, Category). Preview of first 5 rows.
**Step 3 — Validate:** Summary of validation results. Error count by type. Download CSV of failed rows.
**Step 4 — Resolve Duplicates:** List of duplicate phone numbers with per-row actions (Skip/Replace/Keep Both) and bulk action buttons at top.
**Step 5 — Confirm:** Final summary (total to import, categories breakdown). Progress bar during import. Success/partial/error state after completion.

### Add Guest Dialog

- Opens as Dialog (not a new page)
- Fields: Name (required), Phone (required, Bangladesh format validation), Category (select dropdown)
- Phone input shows format hint: "01XXXXXXXXX"
- Validation errors shown inline below each field
- CTA: "Add Guest" (primary button)

## Interaction Patterns

### File Upload Zone (Step 1)
- Drag-and-drop area with dashed border
- Accepts `.csv`, `.xlsx` files only
- Shows upload icon (Upload from lucide) and "Drop your file here or click to browse"
- On file drop: client-side parse begins immediately, zone shows file name + spinner
- After parse: zone shows file name, row count, column count
- Error: zone shows red border with error message

### Column Mapping (Step 2)
- Each detected CSV column shown in a row
- Dropdown selector for target field (Name, Phone, Category, Skip)
- Auto-detection: if column header matches "name", "phone", "category" (case-insensitive), pre-select
- Preview shows first 5 rows with current mapping applied
- "Next" disabled until Name and Phone columns are mapped

### Validation Results (Step 3)
- Summary card at top: "{valid} valid, {errors} with errors"
- Error breakdown table: error type, count, example row
- "Download Error Report" button exports CSV with original data + error column
- If zero errors: step auto-advances or shows green success state

### Duplicate Resolution (Step 4)
- Top bar: "Skip All Duplicates" (outline) + "Replace All with New Data" (outline)
- Table: existing guest data vs new row data, side by side
- Per-row action: radio group (Skip / Replace / Keep Both)
- After bulk action, individual overrides remain possible
- If zero duplicates: step auto-advances

### Guest Table Pagination
- Server-side pagination via Convex cursor-based pagination
- Page size: 50 rows
- Navigation: previous/next arrows + page number input for jumping
- Shows "Showing X-Y of Z guests" below table

### Search and Filter
- Search input is debounced (300ms) — fires Convex search query
- Category filter: Select dropdown populated from event's guest categories
- Status filter: Tabs component with options: All, Invited, SMS Sent, Delivered, Checked In
- Filters combine (AND logic): search text AND category AND status
- URL params sync: filters persisted in URL query params for shareability

## Responsive Breakpoints

| Breakpoint | Layout Changes |
|------------|----------------|
| < 640px (sm) | Filter bar stacks vertically. Table uses horizontal scroll. Import wizard step labels hidden (numbers only). Dialog becomes full-screen sheet. |
| 640-1024px (md) | Two-column filter bar (search + filters row). Table columns prioritized (name, phone, status visible; category in overflow menu). |
| > 1024px (lg) | Full layout as wireframed. All table columns visible. Side-by-side duplicate comparison in Step 4. |
