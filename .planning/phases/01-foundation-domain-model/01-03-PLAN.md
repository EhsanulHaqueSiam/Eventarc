**1. convex/model/vendors.ts** (business logic helpers):

```typescript
import { MutationCtx, QueryCtx } from "../_generated/server";
import { Id } from "../_generated/dataModel";

// Validate vendor category name is unique within its vendor type
export async function validateVendorCategoryName(
  ctx: MutationCtx,
  eventId: Id<"events">,
  name: string,
  excludeId?: Id<"vendorCategories">,
): Promise<void> {
  const existing = await ctx.db
    .query("vendorCategories")
    .withIndex("by_event_name", (q) => q.eq("eventId", eventId).eq("name", name))
    .first();
  if (existing && existing._id !== excludeId) {
    throw new Error(`Vendor category "${name}" already exists for this event`);
  }
}

// Validate stall name is unique within its event
export async function validateStallName(
  ctx: MutationCtx,
  eventId: Id<"events">,
  name: string,
  excludeId?: Id<"stalls">,
): Promise<void> {
  const existing = await ctx.db
    .query("stalls")
    .withIndex("by_event_name", (q) => q.eq("eventId", eventId).eq("name", name))
    .first();
  if (existing && existing._id !== excludeId) {
    throw new Error(`Stall "${name}" already exists for this event`);
  }
}

// Get the full vendor hierarchy for an event (used for sync and display)
export async function getVendorHierarchy(
  ctx: QueryCtx,
  eventId: Id<"events">,
) {
  const vendorTypes = await ctx.db
    .query("vendorTypes")
    .withIndex("by_event", (q) => q.eq("eventId", eventId))
    .collect();

  const categories = await ctx.db
    .query("vendorCategories")
    .withIndex("by_event", (q) => q.eq("eventId", eventId))
    .collect();

  const stalls = await ctx.db
    .query("stalls")
    .withIndex("by_event", (q) => q.eq("eventId", eventId))
    .collect();

  return { vendorTypes, categories, stalls };
}
```

**2. convex/vendors.ts** (public API for vendor categories):

Exports:
- `listByEvent` query: Args = {eventId: v.id("events")}. Returns all vendor categories for the event with their parent vendor type info. Query vendorCategories by_event index, then for each category fetch its vendorType to include the type name ("entry" or "food"). Group result: `{ entry: VendorCategory[], food: VendorCategory[] }`.

- `listTypes` query: Args = {eventId: v.id("events")}. Returns the two vendor types (entry, food) for the event.

- `create` mutation: Args = {eventId: v.id("events"), vendorTypeId: v.id("vendorTypes"), name: v.string()}. Checks auth. Validates event exists. Validates vendor type belongs to event. Calls `validateVendorCategoryName`. Inserts into vendorCategories. Returns categoryId.

- `update` mutation: Args = {categoryId: v.id("vendorCategories"), name: v.string()}. Checks auth. Reads category to get eventId. Calls `validateVendorCategoryName` with excludeId. Updates name.

- `remove` mutation: Args = {categoryId: v.id("vendorCategories")}. Checks auth. Reads category. Deletes ALL stalls under this category first (cascade). Then deletes the category. Returns void.

**3. convex/stalls.ts** (public API for stalls):

Exports:
- `listByCategory` query: Args = {categoryId: v.id("vendorCategories")}. Returns all stalls for the category using by_category index. Ordered by name.

- `listByEvent` query: Args = {eventId: v.id("events")}. Returns all stalls for the event using by_event index.

- `create` mutation: Args = {eventId: v.id("events"), categoryId: v.id("vendorCategories"), name: v.string()}. Checks auth. Validates category belongs to event. Calls `validateStallName`. Inserts with isActive: true. Returns stallId. Each stall gets its own identity (VNDR-02) -- the combination of eventId + categoryId + name makes it unique.

- `update` mutation: Args = {stallId: v.id("stalls"), name: v.string()}. Checks auth. Reads stall to get eventId. Calls `validateStallName` with excludeId. Updates name.

- `toggleActive` mutation: Args = {stallId: v.id("stalls")}. Checks auth. Reads stall. Toggles isActive boolean. This allows admin to deactivate a stall during an event without removing it (VNDR-03).

- `remove` mutation: Args = {stallId: v.id("stalls")}. Checks auth. Deletes the stall document.

All mutations include auth check:

const identity = await ctx.auth.getUserIdentity();
if (!identity) throw new Error("Not authenticated");

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Convex | Frontend connects to Convex via VITE_CONVEX_URL. Auth handled by Better Auth client. |
| Convex -> Go | Internal sync action will POST to Go with HMAC signature (D-14). Stub in Phase 1. |
| Vite proxy -> Go | Dev proxy forwards /api/* to Go server. No auth bypass -- HMAC still required. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Frontend auth | mitigate | ConvexBetterAuthProvider wraps entire app. Unauthenticated users cannot call protected mutations. |
| T-03-02 | Information Disclosure | VITE_CONVEX_URL | accept | Convex URL is public by design (client connects directly). Auth protects data access. |
| T-03-03 | Information Disclosure | VITE_CONVEX_SITE_URL | accept | Auth API URL is public (Better Auth endpoints are designed to be publicly accessible). |
| T-03-04 | Tampering | Vendor hierarchy cascade delete | mitigate | `vendors.remove` deletes stalls first, then category. Transaction safety via Convex mutation atomicity. |
| T-03-05 | Denial of Service | Stall creation | mitigate | `validateStallName` prevents duplicate stall names per event. Convex rate limits protect against spam. |
| T-03-06 | Elevation of Privilege | Vendor mutations | mitigate | All vendor/stall mutations check `ctx.auth.getUserIdentity()`. Unauthenticated calls rejected. |
</threat_model>
