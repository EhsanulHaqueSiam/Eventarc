**2. Create `convex/model/vendors.ts`** -- Business logic helpers:

```typescript
export async function ensureVendorTypes(
  ctx: MutationCtx,
  eventId: Id<"events">,
): Promise<{ entryTypeId: Id<"vendorTypes">; foodTypeId: Id<"vendorTypes"> }> {
  // Check if types already exist for this event
  const existing = await ctx.db
    .query("vendorTypes")
    .withIndex("by_event", (q) => q.eq("eventId", eventId))
    .collect();

  let entryType = existing.find((t) => t.name === "entry");
  let foodType = existing.find((t) => t.name === "food");

  if (!entryType) {
    const id = await ctx.db.insert("vendorTypes", { eventId, name: "entry" });
    entryType = { _id: id, eventId, name: "entry" as const, _creationTime: Date.now() };
  }
  if (!foodType) {
    const id = await ctx.db.insert("vendorTypes", { eventId, name: "food" });
    foodType = { _id: id, eventId, name: "food" as const, _creationTime: Date.now() };
  }

  return { entryTypeId: entryType._id, foodTypeId: foodType._id };
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Vendor/stall mutations | mitigate | All mutations check `ctx.auth.getUserIdentity()`. Unauthenticated calls throw error. |
| T-03-02 | Tampering | Vendor hierarchy integrity | mitigate | Stall deletion only removes the stall. Category deletion cascades to stalls. Event deletion cascades to all children. Convex transactions ensure atomicity. |
| T-03-03 | Information Disclosure | Frontend env vars | mitigate | Only VITE_CONVEX_URL and VITE_CONVEX_SITE_URL exposed to frontend (public by design). No secrets in VITE_ prefixed vars. |
| T-03-04 | Denial of Service | Unrestricted event creation | accept | Phase 1 is admin-only with auth. Rate limiting deferred to production hardening (Phase 10). Low risk with single admin. |
| T-03-05 | Tampering | Client-side form validation bypass | mitigate | All validation duplicated in Convex mutations (server-side). Client validation is UX only. Convex argument validators reject malformed input. |
</threat_model>
