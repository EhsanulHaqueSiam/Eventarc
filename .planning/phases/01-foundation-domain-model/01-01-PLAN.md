**3. backend/internal/config/config.go:**
```go
package config

import "os"

type Config struct {
    Port        string
    Env         string
    DatabaseURL string
    RedisURL    string
    HMACSecret  string
    CORSOrigins []string
}

func Load() *Config {
    env := getEnv("ENV", "development")
    return &Config{
        Port:        getEnv("PORT", "8080"),
        Env:         env,
        DatabaseURL: getEnv("DATABASE_URL", "postgres://eventarc:dev_password@localhost:6432/eventarc?sslmode=disable"),
        RedisURL:    getEnv("REDIS_URL", "redis://localhost:6379"),
        HMACSecret:  mustGetEnv("HMAC_SECRET"),
        CORSOrigins: parseCORSOrigins(env),
    }
}
```
- `getEnv(key, fallback)` reads `os.Getenv` with default
- `mustGetEnv(key)` panics if empty (for HMAC_SECRET)
- `parseCORSOrigins(env)` returns `["*"]` in development, reads `CORS_ORIGINS` comma-separated in production

**6. backend/internal/handler/response.go:**
Helper functions for JSON responses:
- `WriteJSON(w http.ResponseWriter, status int, data interface{})` -- marshals to JSON, sets Content-Type
- `WriteError(w http.ResponseWriter, status int, code string, message string)` -- writes ErrorResponse envelope
- `MapDomainError(err error) (int, string)` -- maps domain errors to HTTP status+code: ErrNotFound->404/NOT_FOUND, ErrDuplicate->409/DUPLICATE, ErrUnauthorized->401/UNAUTHORIZED, ErrInvalidState->422/INVALID_STATE, ErrLimitReached->429/LIMIT_REACHED, ErrAlreadyCheckedIn->409/ALREADY_CHECKED_IN

**7. backend/internal/middleware/hmac.go** (per D-14):
HMAC-SHA256 signature verification middleware. Reads `X-Signature` and `X-Timestamp` headers. Rejects requests older than 5 minutes (replay protection). Computes HMAC over `timestamp + body` using the shared secret. Returns 401 with JSON error on failure. Passes through on success. Body is read and restored via `io.NopCloser`.

**8. backend/internal/middleware/hmac_test.go:**
Tests:
- Valid signature passes through (200)
- Missing X-Signature header returns 401
- Missing X-Timestamp header returns 401
- Expired timestamp (>5 min old) returns 401
- Invalid signature returns 401
- Tampered body fails verification

**9. backend/internal/middleware/logging.go** (per D-25):
Request logging middleware using slog. Logs: request_id (from chi RequestID middleware), method, path, duration_ms, status code. Uses `chimw.WrapResponseWriter` to capture status.

**10. backend/internal/middleware/cors.go** (per D-28):
CORS middleware. In development: allow all origins. In production: allow configured origins only. Allow methods: GET, POST, PUT, PATCH, DELETE, OPTIONS. Allow headers: Content-Type, Authorization, X-Signature, X-Timestamp. Allow credentials: true. Max age: 3600.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Convex -> Go API | Convex HTTP actions call Go endpoints. Must verify HMAC signature to prevent unauthorized sync/scan requests. |
| Browser -> Go API | Vendor scanners will call Go endpoints in later phases. CORS must be correctly configured. |
| Docker network -> Host | Infrastructure services (PG, Redis) exposed on configurable ports. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Go sync endpoint | mitigate | HMAC-SHA256 signature verification in middleware/hmac.go. Shared secret via HMAC_SECRET env var. |
| T-01-02 | Tampering | Go sync endpoint | mitigate | HMAC computed over timestamp+body. Any body modification invalidates signature. |
| T-01-03 | Repudiation | Go sync endpoint | mitigate | Request logging middleware logs request_id, method, path, timestamp for every request. |
| T-01-04 | Replay | Go sync endpoint | mitigate | X-Timestamp header required; requests older than 5 minutes rejected in hmac.go middleware. |
| T-01-05 | Information Disclosure | .env file | mitigate | .gitignore includes .env and .env.local. .env.example has placeholder values only. |
| T-01-06 | Information Disclosure | CORS | mitigate | Production CORS uses explicit allowed origins (no wildcard). Dev allows all origins for local development. |
| T-01-07 | Denial of Service | PgBouncer | accept | PgBouncer limits max_client_conn to 10000. Beyond that, connections are queued/rejected. Acceptable for Phase 1 dev environment. |
| T-01-08 | Elevation of Privilege | Docker services | accept | PG/Redis exposed on localhost only in dev. Production deployment via Dokploy handles network isolation. |
</threat_model>
