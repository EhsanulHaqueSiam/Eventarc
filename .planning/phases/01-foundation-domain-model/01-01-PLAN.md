**1. Create `backend/internal/middleware/hmac.go`** (per D-14):

Implement `HMACAuth(secret string) func(http.Handler) http.Handler` middleware:
- Read `X-Signature` and `X-Timestamp` headers from request
- If either is missing, respond 401 with `{"error":{"code":"UNAUTHORIZED","message":"Missing signature"}}`
- Parse `X-Timestamp` as RFC3339. If invalid or older than 5 minutes, respond 401 with `{"error":{"code":"UNAUTHORIZED","message":"Request expired"}}`
- Read request body into buffer, replace r.Body with `io.NopCloser(bytes.NewReader(body))` so downstream handlers can still read it
- Compute HMAC-SHA256 of `timestamp + body` using the shared secret
- Compare computed HMAC with `X-Signature` header using `hmac.Equal()` (constant-time comparison)
- If mismatch, respond 401 with `{"error":{"code":"UNAUTHORIZED","message":"Invalid signature"}}`
- If valid, call `next.ServeHTTP(w, r)`

Use `crypto/hmac`, `crypto/sha256`, `encoding/hex` from stdlib. Content-Type for error responses: `application/json`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Convex -> Go API | Convex HTTP actions call Go sync/scan endpoints. Must be authenticated via HMAC. |
| Browser -> Go API | No direct browser access to Go API in Phase 1 (only Convex calls Go). CORS configured for future phases. |
| .env secrets | Environment file contains HMAC secret and database credentials. Must not be committed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Go /api/v1/sync/event | mitigate | HMAC-SHA256 middleware on all sync endpoints. Requests without valid X-Signature + X-Timestamp headers are rejected with 401. |
| T-01-02 | Tampering | Go /api/v1/sync/event | mitigate | HMAC signature covers both timestamp and request body. Any modification to the body invalidates the signature. |
| T-01-03 | Repudiation | Go API requests | mitigate | slog request logging middleware logs method, path, request_id, status, duration_ms for every request. |
| T-01-04 | Information Disclosure | .env file | mitigate | .gitignore excludes .env and .env.local. .env.example has placeholder values only. |
| T-01-05 | Denial of Service | Replay attack on Go API | mitigate | X-Timestamp in HMAC payload with 5-minute window. Requests older than 5 minutes are rejected. |
| T-01-06 | Elevation of Privilege | Direct PG access bypassing PgBouncer | accept | Dev environment only. In production, PG port is not exposed; all access goes through PgBouncer. Acceptable risk in dev. |
</threat_model>
