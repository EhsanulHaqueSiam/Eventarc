## Interaction Patterns

### Canvas Editor (Fabric.js)
- On page load: initialize empty Fabric.js canvas with placeholder text "Upload a background image to start"
- Background upload: file picker accepts PNG/JPEG. On select, image loads as canvas background, scaled to fit canvas area
- QR overlay: added automatically after background loads. Sample QR image positioned at center with transform handles
- Drag: click and drag QR overlay to reposition on canvas
- Resize: corner handles on QR overlay for proportional scaling
- Preview: "Preview Composite" button renders canvas to PNG dataURL, shows in Dialog at full resolution
- Save: persists background image URL + QR position (left, top, scaleX, scaleY, angle) to Convex cardTemplates table

### Template Management
- Template list in sidebar shows thumbnail previews (canvas.toDataURL at small resolution)
- Click template to load it into canvas (loadFromJSON restores state)
- "+" button creates new blank template
- Right-click or three-dot menu on template: "Rename", "Delete"
- Active template has accent border highlight

### Compositing Trigger
- "Generate Cards" button disabled until template is saved and QR codes exist (Phase 3 complete)
- Click triggers HTTP POST to Go backend, which enqueues asynq batch job
- Progress updates via polling every 2 seconds (GET progress endpoint)
- On completion: success toast with count, stats cards update

### SMS Delivery Trigger
- "Send Invitations" button disabled until composite cards are generated
- Click shows AlertDialog confirmation: "Send SMS to {count} guests? Standard messaging rates apply."
- On confirm: HTTP POST to Go backend triggers asynq SMS batch
- Progress updates via polling every 5 seconds
- Per-guest status updates in table (real-time via Convex subscription)

### SMS Status Filtering
- Tab bar above delivery table: All / Queued / Sent / Delivered / Failed
- Click tab filters table by status
- Failed tab shows "Retry Failed" button prominently
