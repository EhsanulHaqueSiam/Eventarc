         - `TypeSMSSendBatch = "sms:send:batch"` — single batch of up to 100 numbers

       - Type `SMSWorker struct`:
         - `provider SMSProvider`
         - `redisClient *redis.Client`
         - `asynqClient *asynq.Client`
         - `batchSize int` — default 100
         - `maxBatchesPerSecond int` — default 5
       - Constructor `NewSMSWorker(provider SMSProvider, redisClient *redis.Client, asynqClient *asynq.Client) *SMSWorker`
       - Method `HandleSMSBatch(ctx context.Context, t *asynq.Task) error`:
         - Decode payload: `{ eventId, messageTemplate, guestPhones []GuestPhone }` where GuestPhone is `{ guestId, phone, cardUrl }`
         - Initialize Redis counters: `sms:{eventId}:total={len(guestPhones)}`, `sms:{eventId}:queued={len}`, `sms:{eventId}:sent=0`, `sms:{eventId}:delivered=0`, `sms:{eventId}:failed=0`
         - Split guestPhones into batches of batchSize (100)
         - For each batch: enqueue TypeSMSSendBatch task with batch payload
         - Rate limit: use `time.NewTicker(time.Second / time.Duration(maxBatchesPerSecond))` between enqueues
       - Method `HandleSMSSendBatch(ctx context.Context, t *asynq.Task) error`:
         - Decode payload: `{ eventId, messageTemplate, batch []GuestPhone }`
         - Build message for each guest: replace `{cardUrl}` placeholder in template with guest's card CDN URL
         - Collect phone numbers for batch send
         - Call `provider.Send(ctx, SendRequest{To: phones, Message: message})`
         - If `errors.Is(err, ErrInsufficientBalance)`: set Redis key `sms:{eventId}:balance_error=true`, return error (asynq will not retry since it's a permanent failure — use `asynq.SkipRetry`)
         - On success: INCR Redis `sms:{eventId}:sent` by batch size, DECR `sms:{eventId}:queued` by batch size
         - Store request_id in Redis set: `SADD sms:{eventId}:pending_requests {requestId}`
         - On per-recipient failure: enqueue TypeSMSRetry task with guest info and retryCount=0
       - Method `HandleSMSRetry(ctx context.Context, t *asynq.Task) error`:
         - Decode payload: `{ eventId, guestId, phone, message, retryCount }`
         - If retryCount >= 5: INCR `sms:{eventId}:failed`, DECR `sms:{eventId}:queued`, return nil (permanently failed)
         - Calculate backoff: `delay = 2^retryCount seconds` (1s, 2s, 4s, 8s, 16s)
         - Re-enqueue self with `asynq.ProcessIn(delay)` and retryCount+1
       - Method `HandleSMSStatusPoll(ctx context.Context, t *asynq.Task) error`:
         - Decode payload: `{ eventId }`
         - Read pending request IDs from Redis: `SMEMBERS sms:{eventId}:pending_requests`
         - For each request_id: call `provider.CheckStatus(ctx, requestId)`
         - Count delivered and failed from response
         - Update Redis: INCR `sms:{eventId}:delivered`, INCR `sms:{eventId}:failed` as appropriate
         - Remove completed request_ids from pending set (all recipients have terminal status)
         - If pending_requests still not empty: re-enqueue self with `asynq.ProcessIn(30 * time.Second)`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Go API -> SMS.NET.BD | SMS API key crosses to external provider |
| Admin trigger -> SMS batch | Admin triggers bulk SMS affecting 60K guests |
| SMS.NET.BD -> Go callback | Delivery status from external provider (polled, not pushed) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-09 | Spoofing | POST /sms/send | mitigate | HMAC auth middleware on all SMS API routes |
| T-08-10 | Information Disclosure | SMS API key | mitigate | API key loaded from env var only (SMS_PROVIDER_API_KEY), never logged. HTTP client does not log request params. slog.Info must not include apiKey in structured fields |
| T-08-11 | Denial of Service | Bulk SMS trigger | mitigate | Only 1 active SMS batch per event (check Redis before enqueue, return 409 if already running). Rate limit at 500 SMS/sec to avoid carrier blocks |
| T-08-12 | Tampering | Message template | mitigate | Validate messageTemplate length (max 800 chars per SMS.NET.BD limit). Sanitize — no SMS injection (CRLF chars stripped) |
| T-08-13 | Repudiation | SMS delivery | mitigate | Every SMS send logged with request_id, timestamp, recipient count. Delivery status stored in Convex with full audit trail |
| T-08-14 | Information Disclosure | Guest phone numbers | mitigate | Phone numbers only sent to SMS provider via HTTPS. Never logged at INFO level — only at DEBUG level if needed for troubleshooting |
</threat_model>
