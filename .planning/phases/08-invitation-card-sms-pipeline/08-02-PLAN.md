         - `toPreviewDataURL(): string` — calls canvas.toDataURL({ format: 'png' })
         - `toTemplateJSON(): object` — serializes canvas state via canvas.toJSON()
         - `loadTemplateJSON(json: object): Promise<void>` — restores canvas from JSON via canvas.loadFromJSON()
         - `dispose(): void` — cleanup canvas instance
       - useEffect on mount: initialize `new Canvas(canvasRef.current, { width: 800, height: 600, selection: true })`. Store in state.
       - useEffect cleanup: call dispose on unmount
       - loadBackground: `FabricImage.fromURL(imageUrl)` then set as `canvas.backgroundImage`, call `canvas.renderAll()`
       - addQROverlay: `FabricImage.fromURL(sampleQrUrl)` then `canvas.add(qrImage)`, set as active object, position centered
       - getOverlayConfig: find first FabricImage object (non-background) on canvas, return its left/top/scaleX/scaleY/angle

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file upload -> Canvas | Admin uploads untrusted image files into Fabric.js canvas |
| Frontend -> Convex | Template data and SMS trigger cross to Convex backend |
| Frontend -> Go API | Compositing trigger and progress polling |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Tampering | Image upload | mitigate | Validate file type (PNG/JPEG only via accept attribute), enforce 10MB max size client-side, decode with Fabric.js (rejects malformed images) |
| T-08-07 | Denial of Service | Large image upload | mitigate | 10MB file size limit on upload input, canvas dimensions capped at 4000x4000 |
| T-08-08 | Information Disclosure | Template data | accept | Template data (background URL, QR position) is not sensitive — it is the card design, not guest data |
</threat_model>
