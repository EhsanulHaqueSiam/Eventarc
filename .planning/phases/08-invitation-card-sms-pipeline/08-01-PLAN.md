<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> Go API | Card template data and composite trigger cross from untrusted client |
| Go API -> R2 | R2 credentials used for download/upload of card images |
| Go API -> Convex | Convex deployment token for guest list queries and card URL updates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | POST /cards/composite | mitigate | HMAC auth middleware on all card API routes (already wired via middleware.HMACAuth) |
| T-08-02 | Tampering | Card template data | mitigate | Validate overlay position bounds (left >= 0, top >= 0, width > 0, height > 0) in handler before enqueuing |
| T-08-03 | Information Disclosure | R2 credentials | mitigate | R2 credentials loaded from env vars only, never logged. slog must not log config values |
| T-08-04 | Denial of Service | Composite batch trigger | mitigate | Rate limit composite trigger to 1 active batch per event. Check Redis `composite:{eventId}:total` > 0 before allowing re-trigger |
| T-08-05 | Elevation of Privilege | Convex deployment token | mitigate | Token scoped to specific Convex deployment. Loaded from env only. Never exposed in API responses |
</threat_model>
