Define the QR type constants:
```go
package qr

const (
    PayloadVersion byte = 0x01

    QRTypeEntry   byte = 0x01
    QRTypeFood    byte = 0x02
    QRTypeUnified byte = 0x03
)
```

Define the `Payload` struct:
```go
type Payload struct {
    Version   byte   // Always PayloadVersion
    QRType    byte   // QRTypeEntry, QRTypeFood, or QRTypeUnified
    EventID   string // Convex event document ID (variable length)
    GuestID   string // Convex guest document ID (variable length)
    Timestamp int64  // Unix epoch seconds (creation time)
}
```

Implement `EncodePayload(p Payload, hmacSecret []byte) (string, error)`:
1. Create a `bytes.Buffer`
2. Write `p.Version` (1 byte)
3. Write `p.QRType` (1 byte)
4. Write `byte(len(p.EventID))` (1 byte) — validates length <= 255
5. Write `[]byte(p.EventID)` (N bytes)
6. Write `byte(len(p.GuestID))` (1 byte) — validates length <= 255
7. Write `[]byte(p.GuestID)` (N bytes)
8. Write `p.Timestamp` as uint64 big-endian (8 bytes) using `binary.BigEndian.PutUint64`
9. Compute HMAC-SHA256 over the entire buffer using `crypto/hmac` + `crypto/sha256` with `hmacSecret`
10. Append the 32-byte HMAC signature to the buffer
11. Return `base64.RawURLEncoding.EncodeToString(buf.Bytes())` — this is the QR content string
12. Return error if EventID or GuestID is empty or exceeds 255 bytes

Implement `DecodePayload(encoded string, hmacSecret []byte) (Payload, error)`:
1. `base64.RawURLEncoding.DecodeString(encoded)` to get raw bytes
2. Verify minimum length: at least 1 + 1 + 1 + 1 + 1 + 1 + 8 + 32 = 46 bytes
3. Extract the last 32 bytes as `receivedMAC`
4. Compute HMAC-SHA256 over all bytes except the last 32
5. Use `hmac.Equal()` for constant-time comparison — return `ErrInvalidSignature` if mismatch
6. Parse fields in order: Version (1), QRType (1), EventID length (1) + EventID (N), GuestID length (1) + GuestID (N), Timestamp (8)
7. Validate Version == `PayloadVersion` — return `ErrUnsupportedVersion` if not
8. Validate QRType is one of the three valid values — return `ErrInvalidQRType` if not
9. Return the populated `Payload` struct

Define custom error types:
```go
var (
    ErrInvalidSignature   = errors.New("qr: invalid HMAC signature")
    ErrUnsupportedVersion = errors.New("qr: unsupported payload version")
    ErrInvalidQRType      = errors.New("qr: invalid QR type")
    ErrInvalidPayload     = errors.New("qr: malformed payload")
    ErrEmptyField         = errors.New("qr: empty required field")
)
```

Implement helper `DetermineQRTypes(qrStrategy, foodQrMode string) []byte`:
- If `qrStrategy == "unified"`: return `[]byte{QRTypeUnified}`
- If `qrStrategy == "separate"`: return `[]byte{QRTypeEntry, QRTypeFood}`
- This determines how many QR codes to generate per guest

Implement `GenerateQRImage(content string, cfg GeneratorConfig) ([]byte, error)`:
1. Create QR code: `qrcode.NewWith(content, qrcode.WithEncodingMode(qrcode.EncModeByte), qrcode.WithErrorCorrectionLevel(qrcode.ErrorCorrectionQuart))`
2. Create buffer: `var buf bytes.Buffer`
3. Create writer: `standard.NewWithWriter(nopCloser{&buf}, standard.WithQRWidth(cfg.QRWidth), standard.WithBorderWidth(cfg.BorderWidth), standard.WithFgColorRGBHex(cfg.FgColor), standard.WithBgColorRGBHex(cfg.BgColor), standard.WithBuiltinImageEncoder(standard.PNG_FORMAT))`
4. Save: `qrc.Save(w)` — this writes PNG bytes to the buffer
5. Return `buf.Bytes(), nil`
6. Return wrapped error on any failure

Implement `GenerateGuestQRCodes(eventID, guestID string, qrTypes []byte, hmacSecret []byte, cfg GeneratorConfig) (map[byte][]byte, error)`:
1. `result := make(map[byte][]byte)` — maps QR type to PNG bytes
2. For each `qrType` in `qrTypes`:
   a. Create `Payload{Version: PayloadVersion, QRType: qrType, EventID: eventID, GuestID: guestID, Timestamp: time.Now().Unix()}`
   b. `encoded, err := EncodePayload(payload, hmacSecret)`
   c. `imgBytes, err := GenerateQRImage(encoded, cfg)`
   d. `result[qrType] = imgBytes`
3. Return `result, nil`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| QR Payload HMAC | QR content is HMAC-signed. Any modification to payload bytes invalidates the signature. Scanners (Phase 4) verify before processing. |
| R2 Credentials | AWS-compatible credentials for R2 stored in .env. Never committed to git. Used only by Go worker process. |
| CDN Public URLs | QR images are publicly accessible via CDN. Security relies on HMAC signature in QR content, not URL secrecy (D-07). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | QR payload | mitigate | HMAC-SHA256 signature covers all payload fields. Forging a valid QR requires knowing the secret key. Constant-time comparison via hmac.Equal prevents timing attacks. |
| T-03-02 | Tampering | QR payload bytes | mitigate | Any modification to any byte (version, type, IDs, timestamp) invalidates the HMAC. DecodePayload rejects tampered payloads with ErrInvalidSignature. |
