<objective>
Implement the PostgreSQL durability layer for scan processing: database migration with additional indexes, sqlc-generated query code, asynq background workers for async PG writes and Convex sync-back, and counter re-seeding from PG on Redis restart. This plan ensures that every Redis-committed scan is durably persisted in PostgreSQL via background retries, and that Redis counters can be reconstructed from PG if Redis restarts.
</objective>

Create `backend/cmd/worker/main.go` — standalone asynq worker binary:

```go
package main

// Worker process that handles background tasks:
// - scan:pg-write — persist scan records to PostgreSQL
// - scan:convex-sync — sync check-in status to Convex (placeholder)
//
// Runs as a separate process alongside the HTTP server.
// Config: same env vars as server (DATABASE_URL, REDIS_URL)

func main() {
    cfg := config.Load()
    
    // Connect to PG
    pgPool, err := pgxpool.New(ctx, cfg.DatabaseURL)
    
    // Create PG store
    pgStore := scan.NewPGStore(pgPool)
    
    // Create asynq server
    srv := asynq.NewServer(
        asynq.RedisClientOpt{Addr: redisAddr},
        asynq.Config{
            Concurrency: 10,
            Queues: map[string]int{
                "pg-writes":   6,  // Higher priority
                "convex-sync": 3,
                "default":     1,
            },
        },
    )
    
    // Register handlers
    mux := asynq.NewServeMux()
    mux.HandleFunc(scan.TaskPGWrite, scan.HandlePGWrite(pgStore))
    mux.HandleFunc(scan.TaskConvexSync, scan.HandleConvexSync())
    
    // Start worker
    srv.Run(mux)
}
```

```go
package scan

// ReseedService handles Redis counter re-seeding from PostgreSQL.
type ReseedService struct {
    redis   *redis.Client
    pgStore *PGStore
}

func NewReseedService(redis *redis.Client, pgStore *PGStore) *ReseedService

// ReseedEventCounters re-seeds all Redis counters for a given event from PG data.
// Uses MULTI/EXEC transaction to write all counter values atomically — prevents
// dashboard from reading partial state during re-seed.
//
// Re-seeds: attendance counter, per-category counters, checked-in set.
func (r *ReseedService) ReseedEventCounters(ctx context.Context, eventID string) error

// CheckAndReseed checks if counters exist for an event in Redis.
// If missing (Redis was restarted), triggers ReseedEventCounters.
// Returns true if re-seed was needed and performed.
func (r *ReseedService) CheckAndReseed(ctx context.Context, eventID string) (bool, error)

// ReseedCheckedInSet rebuilds the checkedin:{eventId} set from PG entry_scans.
// Critical for duplicate detection after Redis restart.
func (r *ReseedService) ReseedCheckedInSet(ctx context.Context, eventID string) error
```

The ReseedEventCounters method must:
1. Query PG: `CountEntryScansByEvent` for total attendance
2. Query PG: `CountEntryScansByCategory` for per-category counts
3. Query PG: `SELECT guest_id FROM entry_scans WHERE event_id = $1 AND status = 'valid'` for checked-in set
4. Open Redis MULTI/EXEC pipeline:
   - DEL existing counter hash and checked-in set (clean slate)
   - HSET counters:{eventId} attendance {total}
   - For each category: HSET counters:{eventId} {category}:checkedin {count}
   - For each guest_id: SADD checkedin:{eventId} {guestId}
5. EXEC the pipeline atomically

The CheckAndReseed method must:
1. EXISTS counters:{eventId} — if 0, counters missing
2. SCARD checkedin:{eventId} — if 0 but PG has records, set is missing
3. If either missing: call ReseedEventCounters + ReseedCheckedInSet

Add to Service.ProcessEntryScan:
- Before Lua script execution, call CheckAndReseed if this is the first scan for the event in this server session (use sync.Once per event or similar pattern to avoid checking on every request)

Add admin endpoint in handler: `POST /api/v1/admin/reseed-counters` (HMAC-protected) that accepts `{"event_id": "..."}` and calls ReseedEventCounters.
</action>
