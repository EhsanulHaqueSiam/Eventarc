<objective>
Implement the entry scan processing service with atomic Redis operations. A vendor device POSTs a QR payload to `POST /api/v1/scan/entry`, the handler decodes the QR (reusing `qr.DecodePayload`), validates the HMAC signature, checks guest existence in Redis (with PG fallback stub), and atomically performs check-in via a Redis Lua script that combines duplicate detection, state write, and counter increment into a single atomic operation. Returns structured JSON responses for valid scans, duplicates, and errors.
</objective>

Create `backend/internal/scan/lua.go` containing the Redis Lua script for atomic check-in:

```go
package scan

// checkInScript is the Redis Lua script for atomic entry check-in.
// It performs: (1) check if guest already checked in, (2) if not, mark as checked in,
// (3) store check-in details, (4) increment event attendance counter.
//
// KEYS[1] = checkedin:{eventId}          (SET — checked-in guest IDs)
// KEYS[2] = checkin:{eventId}:{guestId}  (HASH — check-in details)
// KEYS[3] = counters:{eventId}           (HASH — event counters)
//
// ARGV[1] = guestId
// ARGV[2] = timestamp (ISO 8601)
// ARGV[3] = stallId
// ARGV[4] = deviceId
// ARGV[5] = guestCategory (for per-category counter)
//
// Returns:
//   "OK"        — new check-in succeeded
//   "DUPLICATE" — guest already checked in
const checkInLua = `
local already = redis.call('SISMEMBER', KEYS[1], ARGV[1])
if already == 1 then
  return 'DUPLICATE'
end
redis.call('SADD', KEYS[1], ARGV[1])
redis.call('HSET', KEYS[2], 'timestamp', ARGV[2], 'stallId', ARGV[3], 'deviceId', ARGV[4], 'status', 'valid')
redis.call('HINCRBY', KEYS[3], 'attendance', 1)
if ARGV[5] ~= '' then
  redis.call('HINCRBY', KEYS[3], ARGV[5] .. ':checkedin', 1)
end
return 'OK'
`
```

Create `backend/internal/scan/service.go` containing the scan service:

```go
package scan

type Service struct {
    redis    *redis.Client
    pgPool   *pgxpool.Pool
    hmacSecret []byte
}

func NewService(redisClient *redis.Client, pgPool *pgxpool.Pool, hmacSecret []byte) *Service

// ProcessEntryScan is the main scan pipeline:
// 1. Decode QR payload (HMAC verification)
// 2. Validate QR type (must be entry or unified)
// 3. Lookup guest in Redis (HGETALL guest:{eventId}:{guestId})
// 4. If Redis miss: query PG entry_scans for existing check-in, return duplicate if found
// 5. Execute Lua check-in script atomically
// 6. Return ScanResult with guest info + check-in details
func (s *Service) ProcessEntryScan(ctx context.Context, req ScanRequest) (ScanResult, error)

// GetCheckInDetails retrieves the original check-in timestamp and location for duplicate responses
func (s *Service) GetCheckInDetails(ctx context.Context, eventID, guestID string) (CheckInDetails, error)
```

The ProcessEntryScan method must:
- Call `qr.DecodePayload(req.QRPayload, s.hmacSecret)` — returns error if HMAC invalid
- Validate `payload.QRType` is `qr.QRTypeEntry` or `qr.QRTypeUnified` — reject food QR at entry gate
- Build Redis keys: `guest:{eventId}:{guestId}`, `checkedin:{eventId}`, `checkin:{eventId}:{guestId}`, `counters:{eventId}`
- HGETALL guest key for guest info (name, category, photoUrl)
- If guest not in Redis: return `model.ErrNotFound` (PG fallback in Plan 04-02)
- Execute Lua script via `s.redis.Eval(ctx, checkInLua, keys, args)`
- If Lua returns "DUPLICATE": call GetCheckInDetails, return ScanResult with status "duplicate"
- If Lua returns "OK": return ScanResult with status "valid" and guest info

<threat_model>
- **T-04-01 (HIGH):** QR payload forgery — Mitigated by HMAC-SHA256 verification in `qr.DecodePayload()` before any state change. Forged payloads never reach Redis.
- **T-04-02 (HIGH):** Race condition in check-and-set — Mitigated by Redis Lua script executing atomically. SISMEMBER + SADD + HINCRBY in single Lua call, no interleaving possible.
- **T-04-03 (MEDIUM):** Wrong QR type at entry gate — Food QR (type 0x02) at entry gate rejected with 422. Only entry (0x01) and unified (0x03) accepted.
</threat_model>

```go
package scan

// HandleEntryScan returns an HTTP handler for POST /api/v1/scan/entry.
// It decodes the JSON request body, calls Service.ProcessEntryScan,
// and returns the appropriate HTTP status and JSON response.
//
// Status codes:
//   200 — Valid scan (new check-in)
//   409 — Duplicate scan (already checked in) — includes original_scan details
//   400 — Malformed request body or invalid QR payload format
//   401 — Invalid HMAC signature (forged QR)
//   404 — Guest not found in cache
//   422 — Wrong QR type (e.g., food QR at entry gate)
//   500 — Internal server error (Redis/PG connectivity)
func HandleEntryScan(svc *Service) http.HandlerFunc
```

The handler must:
- Decode JSON body into ScanRequest. If decode fails, return 400 with `{"error":{"code":"BAD_REQUEST","message":"Invalid request body"}}`
- Validate required fields: qr_payload non-empty, stall_id non-empty, device_id non-empty. Return 400 if missing.
- Call `svc.ProcessEntryScan(r.Context(), req)`
- Map errors to HTTP status codes:
  - `qr.ErrInvalidPayload` or `qr.ErrUnsupportedVersion` → 400 `INVALID_QR`
  - `qr.ErrInvalidSignature` → 401 `INVALID_SIGNATURE`
  - `model.ErrNotFound` → 404 `GUEST_NOT_FOUND`
  - `qr.ErrInvalidQRType` → 422 `WRONG_QR_TYPE`
  - Other errors → 500 `INTERNAL_ERROR`
- For successful result: if status == "valid" return 200, if status == "duplicate" return 409
- Log scan result with slog: `slog.Info("scan processed", "event_id", eventID, "guest_id", guestID, "status", result.Status, "request_id", chimw.GetReqID(r.Context()))`
</action>
