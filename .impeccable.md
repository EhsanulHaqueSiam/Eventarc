## Design Context

### Users
Corporate event organizers managing large-scale events (up to 60K attendees). They work from office desks during planning/setup phases and switch to tablets on-site during live events. Vendor scanner operators use the scanner UI at physical entry gates and food stalls in outdoor venues with noise, sunlight, and time pressure. Every interaction must feel instant and certain -- no ambiguity about whether a scan succeeded or failed.

### Brand Personality
**Organized. Precise. Confident.**

EventArc is an operations tool that handles high-stakes moments (10K concurrent scans, zero tolerance for errors). The interface should project the same reliability as the system architecture. Users should feel in control of complex, real-time data without ever feeling overwhelmed.

### Aesthetic Direction

**Visual tone**: Monochromatic confidence inspired by Cal.com and Linear. Grayscale palette where boldness comes from contrast and typography, not color. Color is reserved exclusively for semantic states (success green, destructive red, warning amber) and live/active indicators.

**Reference**: Cal.com (monochromatic restraint, Cal Sans typography, multi-layered shadow system, generous whitespace), Linear (dense but organized data, smooth transitions, keyboard-first feel).

**Anti-reference**: Chaotic dashboards with data scattered everywhere. Generic admin templates. Anything that looks like it was assembled from a component library without design intention. Every detail must feel deliberate and over-engineered to perfection.

**Theme**: Light mode only. No dark mode toggle. Pure white canvas with near-black text for maximum contrast and professional confidence.

**Typography**: Cal Sans (display/headings, weight 600, tight tracking at large sizes) paired with Inter (body/UI text). Cal Sans is never used for body text. The font pairing creates a strict division: Cal Sans speaks, Inter explains.

**Elevation**: Multi-layered shadow system (ring shadows as borders + diffused ambient shadows + sharp contact shadows). Avoid CSS `border` for containment -- use the shadow-border technique. Shadows are subtle (5% opacity diffused) with sharp contact edges.

**Scanner UI**: Optimized for outdoor venues with bright sunlight and noise. High-contrast states, large touch targets, haptic feedback on scan results. Success/failure must be immediately obvious from color, size, and vibration -- not just text.

### Design Principles

1. **Monochromatic confidence** -- Grayscale palette with color only for semantic meaning. Boldness through contrast, typography weight, and whitespace, never through decorative color.

2. **Shadow over border** -- Use multi-layered shadow compositing (ring + diffused + contact) instead of CSS borders for card and container elevation. Surfaces should feel like physical objects on a table.

3. **Generous breathing room** -- 80px+ between major sections, tight grouping within related elements. The whitespace is not empty space -- it is the design.

4. **Over-engineered polish** -- Every animation, transition, and micro-interaction must feel intentional. Staggered reveals on page load, smooth state transitions, optimistic UI updates. Nothing should feel like a default.

5. **Scanner-grade clarity** -- The scanner UI must work under pressure: outdoor sunlight, noisy environments, rushed operators. Large targets, high contrast, haptic feedback, unmistakable success/failure states. If an operator has to squint or second-guess, the design failed.
